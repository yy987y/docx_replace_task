# 幼儿园备课包 Word 自动替换项目交接文档

## 项目目标

这个项目用于处理幼儿园保教日志类 Word 模板。核心目标是：

- 读取带有文本框和图形时间轴的 `.docx`
- 识别每个活动块中的一级标签
- 仅替换二级标签和蓝色正文
- 保留一级标签、文本框结构、时间轴图形、字体样式和颜色

当前项目目录：`/Users/yanchao/Documents/develop/docx_replace_task`

## 当前文件说明

- `replace_word.py`
  当前主入口。负责读取 Word、定位文本框中的活动段落、识别一级标签、调用 AI 生成内容，并写出替换后的文档。
- `ai_generator.py`
  真实 AI 生成逻辑。输入一级标签，输出二级标签和蓝色正文。当前通过 HTTP 接口调用模型。
- `replace_word_old.py`
  旧版替换逻辑。它更接近早期 memory 中记录的 run 级精细替换方式，可作为对照参考。
- `原文档.docx`
  当前完整测试输入文档。
- `保教日志_替换后.docx`
  最近一次运行的输出文档。
- `full_process.log`
  历史完整运行日志。
- `test_output.log`
  历史小规模测试日志。
- `tests/run_full_document_check.py`
  新增的完整文档自动化验证脚本，不修改业务代码，直接调用现有主流程。

## 文档结构约定

活动段落的逻辑结构是：

`一级标签：二级标签。蓝色正文`

其中：

- 一级标签保留不变
- 二级标签为黑色正文
- 蓝色正文为蓝色字体

当前 `replace_word.py` 识别的一级标签列表包括：

- 室内自主游戏
- 加点能量
- 早操律动
- 制作
- 主题活动
- 户外体育游戏
- 户外自主游戏
- 起床
- 户外自主游戏回应性照护
- 生活活动
- 抱抱离园
- 餐前活动
- 午餐
- 餐后活动

注意：

- 旧 memory 中曾使用过 `香香进餐`，当前主脚本里已变为 `午餐`
- `replace_word.py` 中的跳过列表仍是 `香香进餐`，如果后续发现跳过逻辑失效，需要优先检查这里

## 运行方式

### 1. 安装依赖

建议使用项目内虚拟环境：

```bash
python3 -m venv .venv
.venv/bin/python -m pip install Spire.Doc requests
```

### 2. 执行业务脚本

```bash
.venv/bin/python replace_word.py
```

默认读取 `原文档.docx`，输出 `保教日志_替换后.docx`。

### 3. 执行完整文档自动化验证

```bash
.venv/bin/python tests/run_full_document_check.py
```

这个脚本会：

- 调用现有 `process_document(...)` 跑真实 AI + 真实 Word 替换
- 输出完整运行日志到 `artifacts/full_document_test.log`
- 检查输出文档是否生成
- 检查输入输出活动段落数量是否一致
- 检查一级标签计数是否一致
- 检查一级标签后是否仍然存在冒号
- 检查替换后内容不是异常短文本
- 检查至少有一个 `★户外自主游戏` 段落含有 `通过...帮助婴幼儿...` 的回应性照护文本
- 如果日志中出现 `AI生成失败`，测试直接判定失败

## 当前实现特点

### `replace_word.py`

当前替换策略是：

- 找到包含一级标签的段落
- 保留一级标签和紧跟的冒号
- 删除后面的所有 run
- 再追加新的二级标签和蓝色正文

这个实现比 `replace_word_old.py` 更简单，优点是流程直观；缺点是对复杂 run 结构的保留能力不如旧版细。

### `ai_generator.py`

生成流程是两段式：

1. 先生成二级标签
2. 再基于二级标签生成蓝色正文

其中做了这些约束：

- 带 `★` 的一级标签要求更长正文
- `加点能量` 会补充点心场景说明
- `★户外自主游戏` 追加回应性照护要求
- 对最近场景做了简单去重

## 已知风险

- `API_URL` 和 `API_KEY` 目前直接写在代码里，存在安全和环境耦合风险
- 历史日志里出现过 `AI生成失败: 'content'`，说明接口返回结构不总是稳定
- 历史日志里也出现过代理连接失败，说明网络环境可能影响运行
- 当前回应性照护依赖模型遵守 prompt，没有程序级二次修正
- 当前主脚本的跳过标签仍是 `香香进餐`，而主识别列表已使用 `午餐`
